# Illumio Blocked Traffic Extractor v1.0.0

## Highlights
- Standalone cross-platform Illumio blocked-traffic extractor with embedded web UI
- Configurable blocked-traffic fetch window instead of a fixed 90-day run
- CSV export plus in-app analytics dashboard and executive-summary view
- CSV re-import for reviewing old exports without rerunning a query
- Shared app theming with `Dark`, `Illumio Dark`, and `Illumio Light`
- Built-in risky-services helper based on Illumio risky service definitions
- Optional live smoke test against a real PCE for API-impacting changes

## Included In This First Release

### Core Extraction
- Query blocked traffic from an Illumio PCE in 1-day chunks
- Filter by:
  - source labels or IPs
  - destination labels or IPs
  - exclusions
  - named PCE services
  - direct protocol/port filters such as `TCP:445`
- Save results to CSV with dynamic label-key columns
- Configure the number of days to fetch

### Hosting and Execution
- Run locally by default on `127.0.0.1:8080`
- Optional hosted/server mode with configurable:
  - host
  - port
  - browser auto-open behavior

### Analytics
- `/summary` dashboard with:
  - port / protocol summary
  - environment cross-talk
  - app-to-app matrix
  - heatmaps
  - top talkers
  - unmanaged / external summaries
- `/executive-summary` dashboard with:
  - summary cards
  - top findings
  - risky service views
  - environment scorecards
  - external spotlight

### CSV Review Workflow
- Import a previously generated CSV on:
  - the main page
  - the analytics dashboard
- Rebuild analytics views from imported CSV data

### Usability and Stability
- Improved cancellation handling
- Improved failed-run completion handling
- Improved discovery error reporting
- Hardened Illumio client JSON parsing
- Added protocol mapping for `IGMP`
- Added show/hide controls for API key and secret entry

### Theming
- Shared theme selection across the full application:
  - `Dark`
  - `Illumio Dark`
  - `Illumio Light`
- Illumio themes aligned to the tracked token reference in `Brand_Tokens.md`

## Known Caveats
- Firefox may still prompt to save credentials because of browser heuristics
- CSV import expects CSVs generated by this tool’s schema
- Live smoke tests require valid PCE credentials and network reachability
