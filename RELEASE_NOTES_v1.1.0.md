# Illumio Blocked Traffic Extractor v1.1.0

## Highlights
- Added a dedicated full-data heatmap explorer with pivot-style filters and click-through drilldown
- Added richer detailed analytics, including an environment service pivot and monthly port/protocol breakdown
- Added explicit extraction date ranges with `Start Date` and `End Date`
- Expanded the executive summary with monthly trends, persistence insights, and change detection
- Fixed named service query encoding so PCE service names work correctly in fetch filters

## Included In This Release

### Analytics Expansion
- Added a dedicated `/heatmaps` page for full environment, application, and combined environment/application heatmaps
- Added pivot-style multi-select filters for heatmaps and drilldown views
- Added click-to-drilldown from heatmap cells into protocol/port composition
- Added collapsible sections to the detailed analytics dashboard
- Added an `Environment Service Pivot` view that behaves more like an Excel-style pivot table for source-environment to destination-environment traffic by protocol and port
- Added a monthly port/protocol breakdown to `/summary`
- Added `Active Connections` to the monthly breakdown so long-lived blocked relationships are visible across month spans without inventing monthly flow volume

### Extraction and Querying
- Added explicit `Start Date` and `End Date` inputs alongside `Days To Fetch`
- Preserved trailing-day extraction as the fallback when no explicit date range is provided
- Fixed named PCE service filters so service names are expanded into Explorer-compatible service tuples instead of rejected HREF objects

### Executive Summary
- Added latest-month trend cards for observed flows, active connections, month-over-month flow change, and new service-pair count
- Added stronger executive findings that incorporate trend and persistence signals
- Added panels for persistent blocked services across months and newly appearing protocol/port pairs in the latest month
- Improved executive scorecards and removed the internal weighted risk score from the visible UI while retaining it for sorting
- Improved card layout so large headline metrics no longer overflow their panels
- Clarified single-month datasets by showing `N/A` for month-over-month flow change when no prior month exists

### Quality and UX Fixes
- Fixed monthly port display formatting so ports such as `5985` are no longer rendered as `5,985`
- Improved theme-awareness and contrast in pivot and executive views
- Kept heatmap dropdown filters open while making multiple selections

## Validation

## Known Caveats
- Firefox may still prompt to save credentials because of browser heuristics
- CSV import expects CSVs generated by this tool's schema
- Live smoke tests require valid PCE credentials and network reachability
