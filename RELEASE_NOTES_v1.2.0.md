# Illumio Blocked Traffic Extractor v1.2.0

## Highlights
- Added configurable extraction chunk intervals from `1 day` down to `5 minutes`
- Added main-page traffic database retention visibility, including server flow days and oldest retained server-flow day
- Improved extractor and analytics layout to use more screen space while simplifying the summary view

## Included In This Release

### Extraction and Visibility
- Added configurable chunk intervals for blocked traffic extraction while preserving final stitched CSV aggregation across the full requested range
- Added a `Traffic DB Availability` panel on the extractor page to load PCE traffic database metrics
- Added a dedicated in-panel trigger for traffic DB metrics instead of overloading the general connection test
- Added an inline progress indicator next to `Load Policy Objects from PCE` so policy-object discovery progress is visible without scrolling to the log pane

### UI and Analytics Workflow
- Widened the extractor, detailed analytics, heatmap explorer, and executive summary layouts to better use desktop screen space
- Removed the redundant embedded heatmap section from the main `/summary` page now that `/heatmaps` is the dedicated heatmap experience
- Reordered `/summary` sections so `Top Talkers` and `Unmanaged / External Summary` appear before the environment and application matrix sections

### Project Metadata
- Added an Apache-2.0 license file to the repository

## Validation
- Verified with `go test ./...`
- Verified with `./smoke_test.sh`
- Rebuilt binaries for Linux, Windows, macOS Intel, and macOS Apple Silicon

## Known Caveats
- Firefox may still prompt to save credentials because of browser heuristics
- CSV import expects CSVs generated by this tool's schema
- Live smoke tests require valid PCE credentials and network reachability
- The app still has no built-in login/authentication layer for hosted multi-user exposure
