# Illumio Blocked Traffic Extractor v1.0.1

## Highlights
- Large-PCE discovery now loads full object collections instead of stopping at the first 500 results
- Discovery collection loading is faster on larger environments through bounded parallelism
- Main-page logging now shows discovery progress directly in the web UI
- Unknown selector values no longer get sent to the PCE as invalid IP addresses
- Extraction reuses the latest discovery cache for the same PCE so large fetches can start promptly after discovery

## Included In This Release

### Discovery and Large-PCE Support
- Switched policy-object discovery to Illumio's asynchronous collection workflow
- Added support for full collection downloads for:
  - labels
  - services
  - IP lists
  - label groups
  - user groups
  - virtual services
  - virtual servers
- Added bounded parallel loading with up to 3 concurrent collection jobs
- Added staged discovery progress messages to the main-page log window

### Extraction Reliability
- Reused the latest successful discovery cache during extraction when the PCE credentials match
- Prevented unknown non-IP selectors from being sent as invalid `ip_address` filters
- Continued to log skipped unknown selectors as warnings for operator visibility

### Validation
- Verified with `go test ./...`
- Verified with `./smoke_test.sh`
- Confirmed successful 90-day extraction behavior against a larger PCE after the discovery and cache changes

## Known Caveats
- Firefox may still prompt to save credentials because of browser heuristics
- CSV import expects CSVs generated by this tool's schema
- Live smoke tests require valid PCE credentials and network reachability
