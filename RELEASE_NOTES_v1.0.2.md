# Illumio Blocked Traffic Extractor v1.0.2

## Highlights
- Improved large-PCE behavior for both connection testing and policy-object discovery
- Fixed summary/dashboard loading after CSV import by disabling stale cached analytics responses
- Preserved the existing analytics import workflow while making it more reliable after restarts or empty-state page loads

## Included In This Release

### Large-PCE UX and Reliability
- Changed **Test Connection** to use a lightweight authenticated API request instead of a full label collection load
- Increased policy-object discovery timeout to 15 minutes for larger PCEs
- Added clearer timeout messaging when discovery exceeds the large-PCE limit
- Prevented failed discovery attempts from leaving stale autocomplete inventory in the UI

### CSV Analytics Review
- Fixed a stale-cache issue where `/summary` could continue showing the empty analytics state after a successful CSV import
- Added explicit no-cache headers to summary and CSV-import analytics responses
- Forced the summary page to refetch the latest analytics state after an import

### Validation
- Verified with `go test ./...`
- Verified with `./smoke_test.sh`
- Verified with a targeted local CSV import and summary reload check against the embedded web app

## Build Artifacts
- `IllumioTrafficTool_Linux`
- `IllumioTrafficTool_Windows.exe`
- `IllumioTrafficTool_MacOS_Intel`
- `IllumioTrafficTool_MacOS_AppleSilicon`

## Known Caveats
- Firefox may still prompt to save credentials because of browser heuristics
- CSV import expects CSVs generated by this tool's schema
- Live smoke tests require valid PCE credentials and network reachability
